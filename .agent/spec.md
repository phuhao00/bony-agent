# Agent 行为规范与工程标准 (Agent Specification)

## 1. 文件系统准则 (File System Rules)

### 1.1 禁止使用系统临时目录
- **核心禁令**: 绝对禁止在任何工具、脚本、环境配置或代码逻辑中使用 `/tmp` 或系统的 `tmp` 目录。
- **替代方案**: 必须使用项目根目录下的 `storage/temp/` 文件夹来存放所有临时文件。
- **路径引用**: 始终使用基于项目根目录的绝对路径或明确的相对路径。

### 1.2 目录结构导向
- 遵循 `project-architect` 技能中定义的标准目录结构。
- 所有的媒体导出、上传素材、持久化数据必须进入 `storage/` 目录。

## 2. 依赖管理准则 (Dependency Rules)

### 2.1 统一环境 (STRICT ADHERENCE REQUIRED)
- **核心原则**: 必须始终使用并维护项目根目录下的 `venv/` 虚拟环境。
- **!!! 严禁!!!**: 绝对禁止对 `venv` 目录进行 `mv` (移动/重命名) 或 `rm` (删除) 操作。
- **严禁重新创建**: 即使遇到环境问题，也必须尝试在该 `venv` 内部进行修复（如安装缺失包），除非用户明确许可，严禁创建任何新的虚拟环境。
- **严禁临时环境**: 绝对禁止为运行单个脚本或测试而创建临时的 `.venv` 或其他命名的虚拟环境。
- **增量更新**: 安装新依赖时，使用 `pip install <package>` 直接更新现有 `venv`，不要执行全量重装（除非必要）。

### 2.2 避免重复下载与长时编译
- **缓存优先**: 充分利用本地 `pip` 缓存。
- **二进制安装**: 安装依赖时优先使用二进制包，避免在本地进行耗时的编译。使用 `pip install --only-binary :all: <package>` 处理易发生编译错误的包。
- **!! 禁止清空 !!**: 除非环境数据物理损坏且无法通过 `pip` 修复，否则严禁删除 `venv` 目录重新构建。
- **快速验证**: 优先检查 `pip show <package>` 确认依赖是否已存在，避免盲目运行安装命令。

## 3. 交互与执行准则 (Execution Rules)

### 3.1 环境一致性
- 所有的 Python 执行必须确保在根目录 `venv` 虚拟环境中运行。
- 启动服务时优先通过 `start_local.sh` 脚本进行，以确保环境变量和路径正确。

### 3.2 自动化与确认
- 对于不涉及文件删除、敏感信息外泄或系统级破坏的命令，可以酌情设置 `SafeToAutoRun: true`，但关键步骤必须征求用户意见。

## 4. 文档与维护准则 (Maintenance Rules)

- **实时更新**: 任何重大的架构调整或规范变更必须同步更新此文件及相关的 `SKILL.md`。
- **原子化提交**: 每次修改应专注于解决一个特定问题，并提供清晰的描述。
